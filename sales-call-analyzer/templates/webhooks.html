Payload Format

All webhooks are sent as POST requests with JSON payloads:

{
  "event": "call.completed",
  "timestamp": "2026-01-07T12:00:00Z",
  "data": {
    "call_id": "abc-123",
    "filename": "sales_call.mp3",
    "status": "complete",
    "score": 85,
    "duration_min": 15.5
  }
}

Signature Verification

Each webhook includes an X-Webhook-Signature header for verification:

X-Webhook-Signature: sha256=abc123...
X-Webhook-Event: call.completed

Verify the signature by computing HMAC-SHA256 of the raw request body with your webhook secret:

# Python example
import hmac
import hashlib

def verify_signature(payload_body, signature_header, secret):
    expected = hmac.new(
        secret.encode(),
        payload_body,
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(f"sha256={expected}", signature_header)

Retry Policy

Webhooks that fail (non-2xx response) are not currently retried. Implement idempotency on your end.
